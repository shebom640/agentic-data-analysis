PROJECT DETAILS
===============
Name: Agentic Data Analysis (Logistics Allocation Suite)
Type: Full-Stack Web Application (FastAPI + Flutter Web)

CORE ARCHITECTURE:
This application replaces black-box LLM systems with a fully deterministic Cascade Fuzzy Logic & Rule-Based Inference System. It is designed to evaluate real-time food-delivery logistics data to determine optimal rider allocation.

1. DATA INGESTION:
The backend utilizes Pandas to process historical CSV datasets (e.g., sales.csv). It extracts critical metrics such as Kitchen Preparation Time (KPT), Rider Waiting Time (RWT), average Order Ratings, and Order Hours. These are normalized into a [0.0 - 1.0] scale to seed the interactive simulation.

INGESTION & QUALITY METRICS:
To guarantee absolute data reliability and transparency before feeding downstream fuzzy reasoning, the Pre-processing Agent computes two explicit metrics:
- NO. OF MISSING VALUES:
  * Definition: The total sum of all null, empty, or unparseable cell values across the entire raw CSV dataset before any processing or cleansing occurs.
  * Purpose: Serves as a baseline metric representing raw dataset quality and data completeness.
  * How it works: Evaluated mathematically using Pandas (`df.isna().sum().sum()`) upon reading the raw upload.
- NO. OF TRANSFORMATIONS:
  * Definition: A cumulative count of all data engineering operations performed to clean, restructure, and coerce the raw data into a reliable, standardized schema.
  * Purpose: Quantifies how heavily the data was scrubbed and imputed, providing a clear explanation trail for auditability.
  * How it works: Increments by 1 for each column renamed during standardization (whitespace stripping, lowercase conversion, space-to-underscore mapping); increments for each row successfully converted to proper Datetime objects; increments by 1 for chronological sorting and 1 for resetting dataframe index; and increments by 1 for each cell in key logistics columns (RWT, KPT, Distance, Hour, Rating) that was null or malformed and had to be coerced/imputed to safe default values.

2. PREDICTION AGENT (Fuzzy Stage 1):
Accepts 'Time of Day' and 'Customer Rating'.
- Fuzzification: Tri/Trap membership functions map values to Morning/Afternoon/Evening and Low/Medium/High sets.
- Rule Base 1 (Service Demand): Maps Time and Rating to an intermediate Service Demand.
- Rule Base 2 (Order Flow): A full 3x3 fuzzy rule base combining the intermediate Service Demand with Time to accurately predict Order Flow.
- Defuzzification: Computes a precise "Order Flow" metric using a weighted centroid mapping.

3. DECISION AGENT (Fuzzy Stage 2):
Accepts the computed 'Order Flow' alongside 'KPT' and 'RWT'.
- Step 1: Fuzzification of normalized KPT, RWT, and Order Flow values.
- Step 2: Rule Base 1 (System State): Evaluates KPT and RWT to determine UNDERUTILIZED, OPTIMAL, OVERLOADED.
- Step 3: Rule Base 2 (Demand Intensity): Fuses System State and Order Flow to determine LOW, MEDIUM, HIGH demand.
- Step 4: Rule Base 3 (Allocation Strategy): Computes REDUCE, HOLD, SCALE UP using a final 3x3 inference stage.
- Step 5: Defuzzification (Centroid Method) and discrete mapping (Reduce: < 0.3, Hold: 0.3-0.7, Scale Up: > 0.7).

All calculations occur with 0 delay at 60fps within the local Flutter application state, guaranteeing highly reactive visual insights for the end user.

TECH STACK & LOGIC OVERVIEW
===========================
1. Frontend (UI & Simulation):
   - Flutter (Dart): Powers the reactive, web-based dashboard for Arsenic Analytics.
   - Core Logic: Calculates entire fuzzy inference models locally within Dart's state loop, providing real-time UI reactions to dataset ingestion.
   - Custom Visualization: Employs Dart's `CustomPainter` to programmatically render fuzzy membership curves (Triangular/Trapezoidal), intersection points, and aggregated strategy boundaries dynamically without bloated external charting libraries.

2. Backend (API & Processing):
   - FastAPI (Python): High-performance routing framework handling file ingestion, backend processing, and acting as the deterministic API layer.
   - Pandas & NumPy: Serves the Data Processing Agent. Parses raw CSVs, standardizes logistics data, performs temporal sorting, and calculates running aggregates (KPT, RWT, Active Orders) instantly upon upload.

3. Artificial Intelligence Paradigm:
   - Deterministic Fuzzy Logic Engine: Replaces non-deterministic LLMs with a 100% explainable mathematical rule-based architecture.
   - Membership Functions: Mathematical functions (`trimf`, `trapmf`) translate crisp normalized values [0.0 - 1.0] into continuous fuzzy degrees (Low, Medium, High).
   - Cascade Inference: Executes sequential, hardcoded boolean rule bases utilizing explicit `MIN` (intersection) and `MAX` (union) operators.
   - Centroid Defuzzification: Computes the geometric center of active fuzzy outputs to synthesize a precise numerical index, mapping seamlessly to rigid decision thresholds (e.g., `< 0.3 REDUCE`).
